# 博客内容

网站框架搭起来之后，就是添加博客内容了，这才是我们搭建网站的初心。

博客内容都应当放置在 docs 目录下，如下：

docs
├── aaa.md
├── bbb 
│   ├── README.md
│   ├── ccc.md
│   └── ddd.md
├── README.md
└── eee.md

# 获取文件访问路径

启动项目之后， vuepress 会把所有的 md 文件编译成 html 文件，并匹配给相应的路径。其中，README.md 会被编译成相应文件夹下的 index.html ，如下所示：

./aaa.html
./bbb/index.html
./bbb/ccc.html
./bbb/ddd.html
./index.html
./eee.html

打开网页后，可直接通过链接访问。

后缀名也可选择不加，则 vuepress 会自动识别，md文件优先于文件夹，例如 localhost:8080/aaa，系统会优先寻找 docs 下的 aaa.md；如果没有，再去找 docs 下 aaa 文件夹下的 README.md 文件。如果仍没有，才跳 404 页面。

# 添加导航栏

通过修改链接访问内容肯定不是一个合格的网站，所以我们需要为其添加导航栏。

在 config.js 中 themeConfig 下的 nav 选项中进行相应配置：

// docs/.vuepress/config.js
module.exports = {
    themeConfig: {
        nav: [
            { text: 'Home', link: '/'},
            { text: '导航一', link: '/bbb/'},
            { text: '外链', link: 'https://jandan.net'}
        ]
    }
}

# 添加侧边栏

导航栏只能作为一个大类，而我们可能会放很多 md 文件，为了添加这些导航，需要使用侧边栏。

同样在 config.js 中 themeConfig 下的 sidebar 选项中配置：

// docs/.vuepress/config.js
module.exports = {
    themeConfig: {
        sidebar: [
            ['/', '首页'],
            ['/bbb/ccc', 'ccc文件'],
            ['/ddd','自定义名称']
        ]
    }
}
